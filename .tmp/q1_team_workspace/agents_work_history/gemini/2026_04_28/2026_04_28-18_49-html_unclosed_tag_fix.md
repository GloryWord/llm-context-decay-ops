# HTML 태그 작성 중 흰색 텍스트 버그 수정

- **작업 일시**: 2026-04-28 18:49
- **작업 목적**: 사용자가 제공한 스크린샷과 같이 HTML 태그를 작성 중이거나 커서가 위치할 때 발생하는 "Unclosed Tag(열린 상태의 태그)" 및 "Linked Editing(매칭 태그 동시 하이라이트)"에서 텍스트가 흰색으로 깨져 안 보이는 현상 수정.
- **작업 내용**:
  - 미완성 태그로 인해 파서가 깨지면서 발생하는 `meta.tag` 및 `invalid.illegal` 스코프에 짙은 녹색(`#3b533e`) 할당.
  - 파서 깨짐으로 인해 속성으로 오인되는 문자열 가독성을 위해 `entity.other.attribute-name` 스코프에 부드러운 브라운(`#8c7b62`) 할당.
  - 커서가 위치할 때 짝꿍 태그를 하이라이트하는 `editor.linkedEditingBackground`, `editor.wordHighlightBackground` 등에 부드러운 올리브 투명색상(`#d0dadd50` 등)을 부여하여 배경이 하얗게 덮이거나 글자가 묻히는 문제 완전 해결.
- **결과**: 태그를 여는 타이핑 순간부터 닫을 때까지 모든 과정에서 HTML 구조의 가독성이 Olive Dunk 테마에 맞게 일관적으로 유지됨.
