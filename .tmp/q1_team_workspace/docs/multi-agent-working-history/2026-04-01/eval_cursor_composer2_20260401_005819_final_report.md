## 항목별 상세

### (1) 데이터 정합성

- **검증한 것**: `experiment_summary.json`의 `condition_means`·`threshold_detection`이 보고서 **§3.1·§3.2**와 **수치상 일치**합니다.  
  - 예: R1 적대 DO/CT **≈4.17턴 → 보고서 T4.2**, R3 적대 CT **≈7.20 → T7.2**, R7 적대 CT **≈7.46 → T7.5** 등.
- **한계**: “29,700 자동 규칙 검사”는 코드베이스에 **동일 산식이 없음**. 응답당 규칙 수가 케이스마다 달라 단순 `10×10,890`이 아님.
- **행동 규칙 API 횟수**: `batch_judge`는 **pending인 `llm_judge` 슬롯마다** 호출하므로, 보고서의 **10,890**이 “모델 응답 수”와 같다면 **평균적으로 케이스당 행동 규칙 1개** 수준으로만 잡혔다는 뜻이고, 그렇지 않다면 **보고서 숫자와 불일치** 가능성이 큼.

**등급: 중** (핵심 표는 요약 JSON과 맞으나, 인프라 요약 수치·judge 건수 정의는 추가 검증 필요)

### (6) 개선 제안 (구체)

1. **메타데이터**: 결과 JSONL 상단 또는 `experiment_summary.json`에 `judge_model`, `vllm_revision`, `pip freeze` 해시, **총 judge 호출 수**를 로그로 남겨 보고서 숫자와 **자동 대조**.
2. **통계**: 셀별 **신뢰구간**, **R3 T10 vs T15** 같은 비단조에 대한 간단한 민감도 분석.
3. **임계값 정의**: 현재 DO/CT는 “각 실행에서 임계 이하로 **처음** 떨어진 턴”의 **평균** — 생존/곡선 기반 정의와 다를 수 있음을 방법에 명시.
4. **코드**: `batch_judge`에서 `rule_id`로 규칙 dict를 찾아 매칭해 **인덱스 가정 제거**.
5. **테스트**: `generate_report.compute_summary`에 **고정 소표 JSON** 단위 테스트.
6. **서술**: Q2는 차트가 “유형별 턴별 평균”이므로, 페르소나·언어 곡선을 **한 문장으로 고정**해 영·한 본 간 불일치 제거.
