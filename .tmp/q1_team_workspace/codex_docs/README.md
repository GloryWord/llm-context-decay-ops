# codex_docs

Codex/OMX 운영 문서를 모아두는 폴더입니다.
여기서 말하는 운영 계약은 **Codex + OMX 기준**이며, maestro 전용 보조 구성은 포함하지 않습니다.
처음 읽는다면 `initial-input-and-final-blocker-only.md`부터 보면 됩니다. 그 문서에 "Codex 터미널에 무엇을 어떻게 입력하는지"가 먼저 적혀 있습니다.

## Files

- `initial-input-and-final-blocker-only.md` — 바쁜 사용자를 위한 가장 짧은 안내. 시작할 때 Codex에 어떻게 말하면 되는지, 막히면 어떤 형식으로 보고가 오는지 설명
- `omx-domain-workflow.md` — 왜 이런 운영 방식을 쓰는지, 작업 종류별로 어떻게 처리하는지 설명하는 상세 문서

## Canonical workflow 

- 최상위 기준 문서는 여전히 `../CODEX.md`입니다.
- 이 폴더는 사람이 읽는 **설명/운영 지식 보관용**입니다.
- `MJ_Codex`를 별도 외부 reviewer/helper로 두는 maestro 전용 해석은 이 폴더의 범위 밖입니다.
- 기준 문서 = 여러 문서가 섞여 있을 때 가장 우선해서 따르는 문서
