3. `CLAUDE.md` 안의 수치 체계가 서로 바로 연결되지 않습니다.  
`Data Flow`에는 “현재 10개 샘플”, `Key File Map`에는 “308 케이스 생성기”, 변수표에는 `rule_count 4수준 × turn_count 4수준 × attack_intensity 2수준 × 반복 5회`가 제시됩니다. 이 숫자들이 어떤 모수, 샘플, 파일 단위를 뜻하는지 설명이 없어 청중이 실험 규모를 오해할 가능성이 큽니다.

4. 이번 작업물만으로는 “연구 질문 → 결과 → 결론”의 완결된 논리 사슬을 검증할 수 없습니다.  
제공된 내용은 설계/운영 문서에 가깝고, 실제 결과표·그래프·결론 문단이 없습니다. 따라서 수치-서술 일치와 과대 해석 여부는 “부분 검토”만 가능합니다.

5. 발표용 문서로는 연구 내용과 내부 운영 규칙이 과도하게 섞여 있습니다.  
`Git Rules`, `Multi-Agent Workflow`, `절대 규칙`은 내부 운영에는 유용하지만, 연구 발표에서는 핵심 메시지를 흐립니다. 청중 입장에서는 연구 설계보다 도구 운용 규칙이 먼저 보이는 구조입니다.
