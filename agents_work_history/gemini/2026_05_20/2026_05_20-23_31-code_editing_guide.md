# 2026년 05월 20일 작업 기록 - 코드 편집 가이드라인 제공

## 1. 개요 및 목적
- 사용자가 질문한 **"hi, how to edit my code?"**에 대해, 본 프로젝트(LLM Context Decay 실험 파이프라인)의 코드 구조와 워크플로우에 기반한 코드 편집 가이드라인을 제공합니다.
- 특히, 본 저장소의 공식 워크플로우 문서인 `CODEX.md` 및 `AGENTS.md`에 명시된 규칙(Codex-first, evidence-first, 4대 Route 분기)을 준수하면서 안전하고 완벽하게 코드를 수정하는 방법을 안내합니다.

## 2. 프로젝트 코드 수정 시 핵심 규칙 (CODEX.md 기반)
본 프로젝트는 **캡스톤 디자인 및 학부 논문용 실험 파이프라인**을 관리하므로, 단순 코드 수정 외에도 실험 결과 검증 및 수치 정합성이 매우 중요합니다.

1. **Codex + OMX 중심의 개발 (Codex-first)**:
   - 본 프로젝트의 1차 구현자(Executor) 및 오케스트레이터는 Codex와 OMX입니다.
   - 외부 AI(Gemini CLI, Cursor 등)는 좁은 범위의 피드백, 논리 검토, 보조 리뷰어로만 활용합니다.
2. **증거 우선주의 (Evidence-first)**:
   - 코드를 수정하기 전, 실제 파일/데이터/설정의 내용을 직접 열어 보고 확인한 뒤에만 수정을 결정해야 합니다. (`이럴 것이다`라는 추측 기반 작업 금지)
   - 코드 수정 후에는 반드시 테스트 명령어(예: `python3 -m pytest`)나 실행 스크립트를 동작시켜 검증 결과를 직접 눈으로 확인해야 합니다.
3. **Route B (Pipeline / Code) 가이드라인 준수**:
   - `src/`, `scripts/`, `tests/`와 관련된 파이프라인 코드를 변경할 때는, 코드가 바뀌는 즉시 관련 문서 경로/파일명/기본 입력 예시가 동기화되어 drift가 발생하지 않는지 검토해야 합니다.
   - 검증 명령: 
     - 의존성 설치: `python3 -m pip install -r requirements.txt`
     - 테스트 실행: `python3 -m pytest`

## 3. 구체적인 코드 편집 방법 (어시스턴트 활용 및 직접 수정)

### A. AI 어시스턴트(Antigravity 등)를 통한 편집
AI 코딩 어시스턴트인 저에게 요청하면 가장 빠르고 정확하게 파일 내용을 수정할 수 있습니다.
- **수정 요청 방법**:
  - "어떤 파일의 몇 번째 줄 근처를 어떻게 수정해 줘" 또는 "어떤 버그를 해결하기 위해 코드를 고쳐 줘"라고 자연어로 명령합니다.
  - 예시: `src/data_pipeline/loader.py 파일의 34번 줄에 있는 예외 처리 로직에 로깅을 추가해 줘.`
- **사용되는 내부 도구**:
  - `replace_file_content`: 파일 내 단일 연속 블록을 교체할 때 사용합니다.
  - `multi_replace_file_content`: 파일 내 여러 개의 흩어져 있는 코드 블록들을 동시에 안전하게 고쳐 쓸 때 사용합니다.
  - `write_to_file`: 완전히 새로운 파일이나 스크립트를 생성할 때 사용합니다.

### B. 사용자가 직접 로컬 IDE(VS Code, Cursor 등)에서 편집
직접 수정할 때도 프로젝트 워크플로우인 `CODEX.md`에 명시된 규칙을 따르는 것이 최선입니다.
1. **대상 파일 확인**: 편집할 파일의 실제 내용을 직접 열어봅니다.
2. **코드 작성 및 빌드**: 로컬 환경에서 코드를 편집하고 저장합니다.
3. **로컬 테스트 실행**: 터미널에서 `python3 -m pytest` 등을 실행하여 정상 동작 여부를 확인합니다.
4. **버전 관리 및 기록**:
   - `GEMINI.md` 및 `CODEX.md`에 의하면, 파일 수정 후에는 반드시 `git add`, `git commit`, `git push`를 진행해 형상을 보존합니다.
   - 작업 후에는 본 파일처럼 `./agents_work_history/gemini/` 경로 밑에 날짜별 마크다운 파일로 작업 내용을 기록합니다.
